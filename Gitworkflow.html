GIT BASICS

  1. Once in our working directory, we can create a new branch (an independent line of develoment) which we can work from by using the "git branch (name of branch)" command.
  2. We can create files using touch then file name OR code then file name to start editing.
  3. We can use the "code" command then the name of the file we wish to edit to modify exisiting files.
  4. After the changes are made, we stage the file using the "git add (file name)".
    Or we can use "git add ." to stage all modified files within the directory.
  5. We can then commit the staged file/s to our LOCAL REPOSITORY using the "git commit -m "messsage" command. The message must be meaningful and relevant.
  6. Now that the local files have been committed, we have moved them from the staged area to our local repository, ready for pushing to remote repository.
  7. We can double check all the changes that have been committed by using the command "git log"
